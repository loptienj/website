Linux and the command line

The default shell

The user's shell (the program that accepts and executes commands) is selected during the account request process. Most users are using the default shell, "bash."

When you log in you may see a basic shell prompt: -bash-2.11 $

You can customize the prompt by setting the PS1 environment variable. Some examples are listed here; but you can find more examples online, or construct your own by consulting the bash manpage.

Black username, Blue host, Green working directory
PS1="[\[\e[0;30m\]\u\[\e[0;31m\]@\[\e[0;34m\]\h \[\e[0;32m\]\W]\$\[\e[0m\] "
A simple blue prompt with the time of day, user@host, working directory:
PS1="\[\033[1;34m\][\$(date +%H:%M)][\u@\h:\w]$\[\033[0m\] "

When a user logs in interactively and starts a Bash shell, a number of scripts are executed automatically. Users typically edit these scripts in their home directory to set up their environment upon login. In the RC environment, edit the .my.xxx version of these files, which is done for portability.

.my.bash_profile
this is executed for login shells
.my.bashrc
executed for non-login shells (thus not re-executed if you invoke a second shell)
.my.logout
executed at logout

In some cases, it may be appropriate to record all configuration in .my.bashrc and "source" .my.bashrc from .my.bash_profile.

It is possible to add modules to your .my.bashrc or .my.bash_profile, but this can be dangerous. In the RC environment it is very important to be aware that your jobs will run in a different environment from the one you set up for your interactive login shell. Your job scripts must be written to load relevant modules as needed.

If you would like to change your shell, please send us an email at rc-help@colorado.edu, and tell us your username and shell of choice.
